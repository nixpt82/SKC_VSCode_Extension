# Business Central Translation Guide

**Complete Multilanguage Workflow for BC Extensions**

---

## Quick Start

### Step 1: Ensure GenerateCaptions is Enabled
Check your `app.json`:
```json
{
  "features": [
    "GenerateCaptions",    // ← Required
    "TranslationFile"      // ← Required
  ]
}
```

### Step 2: Use Label Variables (Not Hardcoded Text)
```al
// ✅ GOOD - Translatable
var
    SuccessMsg: Label 'Operation completed successfully.';
begin
    Message(SuccessMsg);
end;

// ❌ BAD - Not translatable
begin
    Message('Operation completed successfully.');
end;
```

### Step 3: Translate Using bc-translator Subagent
```
"Translate this extension to French, German, and Portuguese"
```

The subagent will automatically:
1. ✅ Build to generate .g.xlf
2. ✅ Create language files
3. ✅ Auto-translate via Azure
4. ✅ Verify 100% completion
5. ✅ Rebuild with translations

---

## What Gets Translated

### ✅ User-Facing Text (Translated):
- Field captions on tables
- Page titles and group captions
- Action captions
- Label variables (Message, Error, Confirm)
- ToolTips
- Enum values
- Option captions

### ❌ Developer Text (NOT Translated):
- Telemetry messages (Locked = true)
- Internal variable names
- Procedure names
- Code comments

## Translation Workflow

### Using bc-translator Subagent:

```
User: "Translate to French"

Subagent automatically:
1. Builds project → Creates .g.xlf
2. Creates French XLF → Companial Interface.fr-FR.xlf
3. Translates → 100% via Azure Translator
4. Verifies → Checks all strings translated
5. Rebuilds → Package includes French
6. Reports → "✅ French translation complete!"
```

### Manual Workflow (If Needed):

```powershell
# Step 1: Build to generate XLF
al_build

# Step 2: Create target language file
createLanguageXlf(
  generatedXlfFilePath: "Translations/MyApp.g.xlf",
  targetLanguageCode: "fr-FR"
)

# Step 3: Translate
skc_translate_xlf(
  sourceFilePath: "Translations/MyApp.g.xlf",
  targetLanguage: "fr-FR"
)

# Step 4: Check status
skc_list_translation_files()

# Step 5: Rebuild with translations
al_build
```

## File Structure

```
Your Extension/
├── app.json                          // Must have GenerateCaptions
├── Translations/                     // Auto-created folder
│   ├── MyApp.g.xlf                  // Source (generated by build)
│   ├── MyApp.fr-FR.xlf              // French translation
│   ├── MyApp.de-DE.xlf              // German translation
│   └── MyApp.pt-PT.xlf              // Portuguese translation
└── src/
    └── ... your AL files ...
```

### File Types:
- **`.g.xlf`** - Generated source file (created by AL compiler)
- **`.{lang}.xlf`** - Target language files (created by you)
- Both are XML/text files and can be edited manually if needed

## Prerequisites

### 1. In app.json:
```json
{
  "features": [
    "GenerateCaptions",    // Generates .g.xlf during build
    "TranslationFile"      // Uses XLF files in package
  ]
}
```

### 2. All User Text Uses Labels:
```al
// ✅ Correct - User-facing message with Label
var
    CustomerNotFoundErr: Label 'Customer %1 not found.';
begin
    Error(CustomerNotFoundErr, CustomerNo);
end;

// ✅ Correct - Telemetry stays in English
var
    TelemetryMsg: Label 'Customer synced', Locked = true;
begin
    Session.LogMessage('00001', TelemetryMsg, ...);
end;
```

### 3. Azure Translation Function (Optional):
- Required for `skc_translate_xlf` auto-translation
- Configure `skc.azureFunctionUrl` in VS Code settings
- Without it: Create XLF files manually and translate yourself

## Best Practices

### Before Translation:

1. ✅ **All user text uses Labels** - Run bc-reviewer to check
2. ✅ **Project compiles** - Fix errors first
3. ✅ **Telemetry in English** - With Locked = true
4. ✅ **Meaningful label names** - Not Label1, Label2

### During Translation:

1. ✅ **Review .g.xlf** - See what will be translated
2. ✅ **Test build first** - Ensure .g.xlf is created
3. ✅ **One language at a time** - Or batch all at once
4. ✅ **Check progress** - Use skc_list_translation_files

### After Translation:

1. ✅ **Review translations** - Technical terms may need adjustment
2. ✅ **Test in BC** - Switch language and verify UI
3. ✅ **Check text length** - Some languages are longer
4. ✅ **Verify ToolTips** - Make sense in each language

## Troubleshooting

### Issue: No .g.xlf file created
**Cause:** GenerateCaptions not in app.json  
**Fix:** Add to features and rebuild

### Issue: Translation tool fails
**Cause:** Azure Function URL not configured  
**Fix:** Set skc.azureFunctionUrl in settings

### Issue: Some strings not translated
**Cause:** Hardcoded text (not Label variables)  
**Fix:** Convert to Labels, rebuild, translate

### Issue: Translation seems wrong
**Cause:** Auto-translation may not understand context  
**Fix:** Manually edit .xlf file with correct translation

## Manual Translation (Without Azure)

If you don't have Azure Translation:

### 1. Create XLF File:
```
createLanguageXlf → Creates empty template
```

### 2. Edit XLF Manually:
```xml
<trans-unit id="Field123">
  <source>Customer Name</source>
  <target>Nom du client</target>  <!-- Add this -->
  <note>Caption for customer name field</note>
</trans-unit>
```

### 3. Save and Rebuild:
```
al_build → Package includes translation
```
